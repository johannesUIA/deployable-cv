Roombooking — Motel Booking System

Aug 2024 – 2024

A PHP-based booking system for a 25-room motel with multiple room types, occupancy limits and admin tooling. Built core booking flows, availability rules and a clean UI to manage rooms and reservations.

Highlights

  • Guest/admin registration with role-based actions
  • Room types (single, double, junior suite) with occupant limits
  • Search by check-in/out date, adults and children
  • Admin tools to manage room types and mark rooms unavailable
  • Optional preferences (floor level, near elevator) + loyalty hooks

Focus areas: server-side PHP and form handling, MySQL schema design, readable functions/arrays, and usability for both motel owners and guests. Designed with the needs of frequent and occasional guests in mind. Code is structured for extension — e.g., stored preferences and stay history to support receipts and loyalty features later.
